SFINAE (Substitution Failure Is Not An Error)
SFINAE - это правило в C++
Согласно нему, если подстановка аргументов в шаблон приводит к ошибке, компилятор не считает это ошибкой, а просто исключает его из списка возможных кандидатов
Это позволяет создавать перегрузки функций и шаблонов, которые работают только с определенными типами

ФАЙЛ 12-2.cpp
std::enable_if - это шаблон из <type_traits>, который позволяет условно включать или отключать перегрузки функций или специализации шаблонов на основе логического условия
Если условие истинно, то std::enable_if предоставляет тип type (по умолчанию void)
Если ложно - шаблон исключается благодаря SFINAE

ФАЙЛ 12-3.cpp
<type_traits> — это заголовочный файл стандартной библиотеки C++, предоставляющий инструменты для анализа и преобразования типов данных во время компиляции
Type Traits - это шаблоны из <type_traits>, которые представляют информацию о типах на этапе компиляции
Примеры:
1.std::is_integral<T> - проверяет, является ли T целочисленным типом
2.std::is_pointer<T> - проверяет, является ли T указателем
3.std::remove_const<T> - удаляет const из типа
4.std::conditional<Condition, T, F> - выбирает между T и F в зависимости от выполнения условия

С C++20 вместо SFINAE и std::enable_if стали использоваться концепты, которые упрощают условную компиляцию

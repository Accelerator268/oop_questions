Итератор - объект, действующий как мост между алгоритмами STL и контейнерами, обеспечивающий унифицированный доступ к элементам без знания структуры данных

В моем коде *it получает значение, ++it переходит к следующему элементу. Итератор может работать с разными типами контейнеров.
Реализованы:
1. Инициализация current (текущее значение) значением pos
2. Оператор разыменования (возвращает текущее значение)
3. Префиксный инкремент (переводит к следующему значению)
4. Оператор 

К последовательным контейнерам относятся vector (динамисеский массив), deque (двусторонний массив), list (двусвязный список), array (фиксированный массив) и forward_list (односвязный список)
Они упорядочено хранят элементы, позволяя управлять их позицией

К ассоциативным контейнерам относятся set, map, multiset, multimap. Они хранят данные парами "ключ-значение"
Map это словарь, хранит уникальные пары ключ-значение, отсортированные по ключу
Set хранит только множество уникальных ключей, отсортированных по возрастанию

Виды итераторов:
1. Input (ввод). Считывание элементов раз за проход, ++ для движения вперед. Пример isrteam_iterator
2. Output (вывод). Запись элементов раз за проход, ++ для движения вперед. Пример ostream_iterator
3. Forward (однонаправленные). Считывание/запись, возможность многократного прохода вперед (++), запоминание позиции. Пример forward_list
4. Bidirectional (двунаправленные). То же, что и прошлый, только можно двигаться еще назад (--). Пример list, set
5. Random Access (произвольный доступ). Имеет максимальные возможности: доступ к элементу по индексу, арифметические операции и тд. Пример: vector, deque

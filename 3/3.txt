ФАЙЛ 3-1.cpp
Исключение - механизм обработки ошибок в C++. Они позволяют выбрасывать и ловить ошибки
В них используются throw для вызова ошибки и catch для ее поимки
Также используется try для обозначения блока кода, в котором может выброситься ошибка

ФАЙЛ 3-2.cpp
std::exception - базовый класс для всех стандартных исключений, имеет метод what() для возврата описания ошибки
std::exception_ptr - умный указатель на исключение. Позволяет сохранить исключение и повторно выбросить его позже
std::current_exception - возвращает std::exception_ptr на текущее исключение
std::rethrow_exception - повторно выбрасывает исключение, сохраненное в std::exception_ptr

ФАЙЛ 3-3.cpp
В catch есть перехват по типу и перехват всех исключений

ФАЙЛ 3-4.cpp
Модификатор noexcept сигнализирует о том, что функция не выбрасывает исключений
Если помеченная функция все же выбросит исключение, то прога завершится с вызовом std::terminate

Плюсы использования исключений:
1.Упрощают обработку ошибок, особенно в глубоких вызовах функций
2.Позволяют отделить код обработки ошибок от основного кода
3.Поддерживают иерархию исключений (наследование от std::exception)

Минусы использования исключений:
1.Могут быть медленными (особенно, если исключения выбрасываются часто)
2.Усложняют понимание потока выполнения программы
3.Не подходят для обработки ошибок в реальном времени
